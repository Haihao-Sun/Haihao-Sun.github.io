Author: Haihao Sun Date: September 6, 2020  20:13:42 Category: CPP

1. Reference and Pointer

  • reference cannot be empty, but pointer could potentially be nullptr
  • reference cannot be reassigned to another value, but we can use pointer to point to another object
  • destroy, create reference will not call the destructor and constructor
  • *ptr ≈ ref

Pointer has its value, and memmory address (double pointer)

2. Virtual and Pure Virtual Function

Do we need to override (pure) virtual function in cpp?

  • We can override Virtual Function, and we can also choose not to, but if we do not override the virtual function, the child class will use the virtual function in the parent class.
  • We HAVE TO override pure virtual function, and we cannot create object for class that has pure virtual class since this function is not defined in this class.

2.1 How to decide which function to call in polymorphism

  • pointer type decides the usage of normal function
  • object type decides the usage of virtual function

2.2 What is the memory layout for virtual function and pure virtual function?

text area : foo(), bar() -> this part is owned by all objects in this class
stack: val -> this part is private to that specific object

text area: bar() foo() -> they are not stored in the same neighbouring location
stack: val, vptr -> vptr points to a virtual table which contains a function pointer pointing to foo() that we override
global: *(foo())

2.3 Can we make constructor to be virtual? What about making destructor to be virtual?

  • We cannot make constructor to be virtual: constructor creates the instance, but if we do not have instance field vptr, we can not find the virtual constructor
  • We might recommend making destructor, since different children class will have distinct instance field, therefore it would be great to implement destructor that is specifically useful for that child class.

3. Const

int a = 10;

  • situation 1: we cannot change a, but we can reasign the pointer if const is at the left of *
  • situation 2: we can change a, but cannot reassign the pointer if const is at the right of *
  • Situation 3: we cannot do both if there are two consts

4.Smart Pointer

What is smart pointer, and why we need it?

Oftentimes, it is very common to forget to delete the object that we just new before. Additionally, there might be errors or exception in the function that we call after use new keyword. Therefore, we can use smart pointer to simulate the garbage collector in Java!

Just Make your C++ look like Java.

Four Common Smart Pointer

  • unique_ptr: cannot be copied
  • shared_ptr: use Reference counting to manage the garbage, can be copied
  • weak_ptr : solve the problem for cycle dependecy
