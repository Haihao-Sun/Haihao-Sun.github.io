Leetcode-98-Validate BST

Problem Description

Given a binary tree, determine if it is a valid binary search tree (BST).

Assume a BST is defined as follows:

  • The left subtree of a node contains only nodes with keys less than the node’s key.
  • The right subtree of a node contains only nodes with keys greater than the node’s key.
  • Both the left and right subtrees must also be binary search trees.

Problem Discussion:

While one possible is to do a pre-order traversal, and we are supposed to get the strictly increasing sequence. However, we can also do this in O(logn) in terms of space complxity (the height of the tree is the recursion depth).

Since tree is recursively defined, we can recursively solve this problem as well. When we visit the node, aka, doing the DFS, we have to remember the curMax and curMin limit for the current node to validate the binary search tree!

The key idea is to remember both the curMax and curMin since if we only remeber one of them, it will violate the BST as given in the graph. Even though 4 > 3 and 4 < 6, but we observe 3 is smaller than 5, whicch means that 3 cannot be at 5's right. Therefore, we need to account for this, and we need to use the program to remember this.
