newcoder-string-transform

Author: Haihao Sun Date: August 28, 2020  22:00:50 Category: Greedy

Problem Description:

Given a string a, string b, and you have 3 options to conduct transformation. Return the minimum number of steps to transform string a to string b. Here are the 3 operations that you can perform.

  • swap A[i] with A[j]
  • flip A[i] from 0 to 1 or from 1 to 0
  • Reverse string A

Problem Discussion

Welcome Back! It has been a while since the last time that I did some coding, and I was spending time implementing EM algorithm last week.


This question looks like a BFS at the first glance, but we know that it is very hard to complete the searching since the searching states(node) and transistion steps(edges) are extremely large, and we need to do something else.


  • We name the first operation swap(i, j) -> S(i, j)
  • We name the second operation flip(i) -> F(i)
  • we name the third operation reverse -> R

The property that we observe is that we can do reverse, and swap, or maybe swap index(a.length - i, a.length - j), and then reverse

The sequence of the operations do not really matter.
Therefore, we can always reschedule our operation in a more formal and logic way. This may remind you that whenever we can change the sequence of the operations, we will have a greedy solution.

Actually, this problem indeed has a greedy solution.
